# Template UNV NVR for Zabbix 7.4

Мониторинг регистраторов Uniview (UNV NVR) через LightAPI с автоматическим обнаружением камер, созданием отдельных хостов камер, инвентаризацией оборудования и автоматическим построением карт сети.

## Возможности

### Мониторинг регистратора

Шаблон `Template UNV NVR` получает:

* Модель регистратора
* Серийный номер
* Версию прошивки
* Hardware ID
* Тип устройства
* Производителя
* Device Code
* Proto Name
* Версию UBOOT

### Контроль изменений

Генерируются события при:

* изменении прошивки регистратора;
* изменении имени регистратора;
* изменении идентификационных данных устройства;
* невозможности получить данные с регистратора;
* изменении карты камер.

### Автоматическое обнаружение камер

Шаблон автоматически:

* получает список подключенных камер через LightAPI;
* создаёт отдельные хосты камер через LLD;
* назначает шаблон `Template UNV Camera`;
* создаёт интерфейс камеры с её IP-адресом;
* заполняет инвентаризацию камеры;
* автоматически удаляет устаревшие камеры спустя 90 дней после исчезновения.

### Мониторинг камер

Для каждой камеры отображаются:

* статус Online/Offline;
* MAC-адрес;
* модель;
* серийный номер.

Создаётся триггер:

* `Camera offline`

### Автоматические зависимости

Для каждой обнаруженной камеры автоматически создаётся зависимость:

```text
Camera offline
    ↓
Can't update info from NVR
```

Если регистратор недоступен, не возникает десятков ложных тревог по всем подключенным камерам.

### Автоматическое построение карт

Для каждого регистратора создаётся отдельная карта сети.

На карту автоматически добавляются:

* регистратор;
* все обнаруженные камеры.

При удалении или переносе камеры карта автоматически обновляется.

# Первоначальная настройка

## 1. Импорт шаблонов

Импортируйте шаблон в Zabbix.

Будут созданы:

### Template groups

```text
Templates/Video surveillance
```

### Host groups

```text
Cameras
Template UNVIEW NVR
```

---

## 2. Создание API-токена

Шаблон использует API Zabbix для:

* получения данных от регистратора;
* создания зависимостей триггеров;
* создания и обновления карт.

Создайте отдельного пользователя.

### Рекомендуемый пользователь

```text
unv-template
```

### Роль

Можно использовать:

```text
Super admin
```

или создать отдельную роль с разрешением на:

* API;
* чтение хостов;
* изменение триггеров;
* чтение элементов данных;
* создание и изменение карт.
(host.get, item.get, trigger.get, trigger.update, map.get, map.create, map.update)


После создания пользователя:

```text
Administration → Users → API tokens
```

Создайте токен.

---

## 3. Настройка макроса API

В шаблоне заполните:

```text
{$TEMPLATE_UNVIEW_NVR.API_TOKEN}
```

Значение:

```text
<ваш API token>
```

Этот макрос обязателен.

Без него не будут работать:

* карты;
* зависимости триггеров;
* получение данных камер из регистратора.

---

## 4. Создание хоста регистратора

Создайте новый хост.

Пример:

```text
NVR-MAIN
```

Подключите шаблон:

```text
Template UNV NVR
```

---

## 5. Настройка макросов регистратора

Для каждого регистратора заполните:

### Логин

```text
{$NVR.USERNAME}
```

Пример:

```text
admin
```

### Пароль

```text
{$NVR.PASSWORD}
```

### Схема подключения

```text
{$NVR.SCHEMA}
```

Варианты:

```text
http
```

или

```text
https
```

---

## 6. Настройка группы камер

По умолчанию:

```text
{$CAMERA.GROUP}=Cameras
```

Все обнаруженные камеры будут автоматически помещаться в эту группу.

Можно указать любую существующую группу.

Например:

```text
Video/Cameras
```

---

## 7. Настройка иконок карт

При необходимости укажите ID иконок:

### Камера

```text
{$CAMERA.ICON.DEFAULT}
```

По умолчанию:

```text
189
```

### Регистратор

```text
{$NVR.ICON.DEFAULT}
```

По умолчанию:

```text
16
```

---

# После запуска

После первого успешного опроса:

1. Будет получена информация о регистраторе.
2. Будут обнаружены камеры.
3. Будут автоматически созданы хосты камер.
4. Будут заполнены данные инвентаризации.
5. Будет создана карта регистратора.
6. Будут настроены зависимости триггеров.

Дополнительных действий администратора не требуется.
